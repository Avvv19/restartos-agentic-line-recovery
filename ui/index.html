Restart OS

Agentic Line-Recovery for Manufacturing

When a production line goes down, a senior maintenance tech spends 45–90 minutes chasing data across 8 different systems just to write the work order. Restart OS does that job — it triages the fault, grounds the procedure in the OEM manual, picks a qualified technician, drafts the work order, reserves the parts, and hands the human a single approve/reject button.

🌐 Live endpoints

Everything the engine exposes is JSON. Each endpoint is documented and audit-logged.

  • GET /cockpit — operator UI
  • GET /api/latest_run — last incident's full trace
  • GET /api/memory — Postgres stats
  • GET /api/rag?q=… — semantic search
  • GET /api/providers — LLM availability
  • GET /api/boundary — OT-write boundary proof
  • GET /healthz — liveness probe
  • GET /metrics — Prometheus exposition
⏱ Production calibration window — 3 to 4 weeks. Restart OS will not autonomously dispatch work orders on a real line until it has spent 3–4 weeks in shadow mode on that plant's own incidents, with a maintenance lead reviewing every proposal. Only when the measured human–agent agreement rate is sustained at ≥ 85% does the gate flip to autonomous. Read the calibration guide →
